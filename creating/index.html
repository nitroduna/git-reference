Obteniendo ya Creando Proyectos

Para hacer cualquier cosa en Git, es necesario tener localmente un repositorio Git. Es donde Git almacena los datos de los snapshots que guardamos.

Principalmente hay dos formas de obtener un repositorio Git. Una es implemente inicializando un nuevo repositorio en un directorio existente, ya sea o un proyecto nuevo o un proyecto existente que se añade al sistema de control de código fuente. La segunda forma es clonarlo desde un repositorio Git público, lo que haríamos si quisiéramos una copia o quisiéramos trabajar con otra gente en algún proyecto. Cubriremos ambos sistemas a continuación.

docs   book git init inicializa un directorio como repositorio Git

Para crear un repositorio desde un directorio existente (que puede contener ficheros), simplemente ejecutamos git init en ese directorio. Por ejemplo, digamos que tenemos un directorio conteniendo unos pocos ficheros, como este: 
$ cd konichiwa
$ ls
README   hello.rb
Es un proyecto en el que estamos escribiendo ejemplos del "Hello World" en todos los lenguajes. Hasta ahora, sólo hemos usado Ruby, pero bueno, es un comienzo. Para empezar el control de versiones de este proyecto con Git, simplemente ejecutamos git init. 
$ git init
Initialized empty Git repository in /opt/konichiwa/.git/
Observaremos que ahora hay un subdirectorio .git en nuestro proyecto. Es nuestro propio repositorio Git donde todos los datos de los snapshots del proyecto son almacenados.. 
$ ls -a
.        ..       .git     README   hello.rb
Enhorabuena, ahora tenemos un esqueleto de repositorio Git, y podemos empezar a crear snapshots de nuestro proyecto.

En definitiva,, usamos git init para hacer que el contenido de un directorio exista en un nuevo repositorio Git. Podemos hacer esto en cualquier directorio y en cualquier momento, de forma completamente local.

docs   book git clone copia un repositorio git para agregarle cosas nuevas

Si necesitamos colaborar con gente en un proyecto, o si queremos obtener una copia de un proyecto para ver o usar el código, tendremos que clonarlo. Simplemente ejecutamos el comando git clone [url] con la URL del proyecto que queremos copiar. 

$ git clone git://github.com/schacon/simplegit.git
Initialized empty Git repository in /private/tmp/simplegit/.git/
remote: Counting objects: 100, done.
remote: Compressing objects: 100% (86/86), done.
remote: Total 100 (delta 35), reused 0 (delta 0)
Receiving objects: 100% (100/100), 9.51 KiB, done.
Resolving deltas: 100% (35/35), done.
$ cd simplegit/
$ ls
README   Rakefile lib

Esto copiará el histórico completo de ese proyecto para tenerlo de forma local y proporcionará un directorio de trabajo (working directory) de la rama (branch) principal de ese proyecto, de forma que podamos ver su código o empezar a modificarlo. Si accedemos al nuevo directorio, podremos ver el subdirectorio .git - que es donde están todos los datos del proyecto. 

$ ls -a
.        ..       .git     README   Rakefile lib
$ cd .git
$ ls
HEAD        description info        packed-refs
branches    hooks       logs        refs
config      index       objects

Por defecto, Git crea un directorio con el mismo nombre que el proyecto en la URL proporcionada - básicamente cualquier cosa que siga a la última barra de la URL. Para usar un nombre diferente, simplemente se añade justo al final del comando despues de la URL.

En definitiva, usamos git clone para obtener una copia local de un repositorio Git de forma que podamos examinarlo o empezar a modificarlo.
