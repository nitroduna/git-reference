Obtener y Crear Proyectos

Para cualquier cosa que se quiera hacer en Git, es necesario obtener un repositorio Git. Es donde Git almacena los datos de los snapshots que salvaguardamos.

Hay dos formas principales de obtener un repositorio Git. Una de ellas tan sólo consiste en inicialiar un nuevo repositorio en un directorio ya existente, ya sea para un proyecto nuevo o un proyecto que se quiere añadir al sistema de control de código fuente. Otra forma es clonarlo desde un repositorio Git público, que es lo que haríamos si quisiéramos una copia o quisiéramos trabajar en algún proyecto colaborativo. Ambos sistemas serán cubiertos en este apartado.

docs   libro git init inicializa un directorio como repositorio Git

Para crear un repositorio desde un directorio ya existente, simplemente ejecutamos git init en ese directorio. Por ejemplo, supongamos que tenemos un directorio que contiene algunos ficheros, como este: 
$ cd konichiwa
$ ls
README   hello.rb
Se trata de un proyecto en el que estamos escribiendo ejemplos del programa "Hello World" en todos los lenguajes de programación. Hasta ahora, sólo hemos usado Ruby, pero bueno, es tan sólo un comienzo. Para iniciar el control de versiones de este proyecto con Git, simplemente ejecutamos git init. 
$ git init
Initialized empty Git repository in /opt/konichiwa/.git/
Ahora podemos observar que hay subdirectorio .git en nuestro proyecto. Se trata de nuestro propio repositorio Git donde todos los datos de los snapshots del proyecto serán almacenados. 
$ ls -a
.        ..       .git     README   hello.rb
Enhorabuena, ya tenemos el esqueleto de nuestro repositorio Git y podemos comenzar la creación de los snapshots de nuestro proyecto.

En resumidas cuentas, usamos git init para convertir un directorio y su contenido en un nuevo repositorio Git. Esto se puede hacer en cualquier directorio y en cualquier momento, de forma totalmente local.

docs   libro git clone copia un repositorio git con el que trabajar

Si necesitamos colaborar con otros en un proyecto, o queremos obtener una copia de un proyecto de forma que podamos examinar o usar su código fuente, será necesario clonarlo. Simplemente ejecutamos el comando git clone [url] con la URL del proyecto que queremos copiar. 

$ git clone git://github.com/schacon/simplegit.git
Initialized empty Git repository in /private/tmp/simplegit/.git/
remote: Counting objects: 100, done.
remote: Compressing objects: 100% (86/86), done.
remote: Total 100 (delta 35), reused 0 (delta 0)
Receiving objects: 100% (100/100), 9.51 KiB, done.
Resolving deltas: 100% (35/35), done.
$ cd simplegit/
$ ls
README   Rakefile lib

Esto copiará el histórico completo de ese proyecto para tenerlo de forma local, y proporcionará un directorio de trabajo (working directory) de la rama (branch) principal de ese proyecto, de forma que podamos examinar su código o empezar a modificarlo. Si accedemos a este nuevo directorio, podremos ver el subdirectorio .git - que es donde están todos los datos del proyecto. 

$ ls -a
.        ..       .git     README   Rakefile lib
$ cd .git
$ ls
HEAD        description info        packed-refs
branches    hooks       logs        refs
config      index       objects

Por defecto, Git crea un directorio con el mismo nombre que el proyecto de la URL proporcionada - básicamente cualquier cosa que siga a la última barra de la URL. Para usar un nombre diferente simplemente se añade como un argumento al final del comando, despues de la URL.

En resumidas cuentas, usamos git clone para obtener una copia local de un repositorio Git, de forma que se pueda examinar o empezar a modificar.
